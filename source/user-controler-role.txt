Вот список всех эндпоинтов, которые ты реализуешь в UserController. Они разделены по методам HTTP и описаны с краткими пояснениями:

1. GET /users — Получение всех пользователей
Описание: Возвращает список всех пользователей.
Доступ: Только администраторы.
2. GET /users/:id — Получение информации о конкретном пользователе
Описание: Возвращает публичную информацию о пользователе по его ID.
Доступ: Обычные пользователи могут получить свои данные, администраторы — любого пользователя.
3. POST /users — Создание нового пользователя
Описание: Создает нового пользователя.
Доступ: Только администраторы.
4. PUT /users/:id — Обновление данных пользователя
Описание: Обновляет информацию о пользователе по его ID.
Доступ: Администраторы могут обновлять любого пользователя, обычные пользователи — только свои данные.
5. DELETE /users/:id — Удаление пользователя
Описание: Удаляет пользователя по его ID (или деактивирует).
Доступ: Только администраторы.


Полный список эндпоинтов:
HTTP метод	Эндпоинт	Описание	Доступ
GET	/users	Получение всех пользователей	Только администраторы
GET	/users/:id	Получение информации о пользователе	Администраторы или пользователь
POST	/users	Создание нового пользователя	Только администраторы
PUT	/users/:id	Обновление данных пользователя	Администраторы или пользователь
DELETE	/users/:id	Удаление пользователя	Только администраторы
Это полная структура эндпоинтов для UserController с учетом ролей и прав доступа.
